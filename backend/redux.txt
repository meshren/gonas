React redux
action描述了如何去修改state,而reducer则是根据action的描述，真正地去修改state
reducer代码量太大，所以要进行拆分，再用combineReducer进行合并，拆分原则是以state中的子元素为基准进行拆分
store是个粘合剂，将redux中的各个概念粘合到一起，createStroe接收reducer为参数，创建store对象
store中提供dispatch，来发送action
store中提供getState，来获取store中全局state
store中提供subscribe，来定义全局的变化

react-redux包的作用：
向根组件注入store->Provider组件
连接React组件和Redux状态层 -> connect
获取React组件所需的State和Actions-> map api
